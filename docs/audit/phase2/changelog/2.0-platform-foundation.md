# Changelog 2.0 — Core platform foundation

## Improvement
- Introduced Application Core + Platform API with ADRs and budget contract.
- Dashboard renders from hydrated cache; refreshes no longer clear cards.
- Realtime uses targeted invalidation; wallet shares trading socket.
- Health probes fail fast.

## Tradeoffs
- Dual L2 storage keys remain behind CacheEngine facade until v2 blob migration.
- Some pages still call `lib/api` directly (incremental).

## Residual risk
- Compat suite needs AUDIT_JWT for authed routes in CI.
- Streaming MetaApi still absent — syncStatus=degraded is the UX safety valve.
