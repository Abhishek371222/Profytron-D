# API Reference

Interactive OpenAPI docs are generated by Nest Swagger in **non-production** environments:

```text
GET http://localhost:4000/api/docs
```

Swagger is disabled in production to avoid advertising endpoint signatures.

## Package overview

See `apps/api/README.md` for local run commands and module folders.

Primary module areas:

| Module path | Domain |
|-------------|--------|
| `auth` | Login, OAuth, JWT, sessions, MFA-related flows |
| `users` | User profile and account |
| `trading` | Trading operations + WebSocket gateway |
| `copy` / `copy-bridge` / `copy-factory` | Copy trading |
| `marketplace` | Strategy marketplace |
| `payments` / `wallet` | Billing, deposits, withdrawals |
| `analytics` | Performance and trade analytics |
| `notifications` | In-app / push dispatch |
| `admin` | Admin operations |
| `ai` / `ai-risk` | AI coach and risk helpers |
| `broker` / `broker-integration` | Broker connectivity |
| `strategies` / `strategy-builder` | Strategy management |
| `websocket` | WS module wiring |

## Shared types

Cross-cutting enums and interfaces live in `@profytron/types` (`packages/types`).

## Health

```text
GET /health
```

Used by load balancers and deploy probes. See [RUNBOOK.md](./RUNBOOK.md).

## Auth notes

- Bearer access tokens and cookie refresh (`refresh_token`) are documented in Swagger when enabled.
- Web OAuth callback: `apps/web/src/app/(public)/auth/callback/`
- Configure Google/Supabase OAuth via web and API env templates (`.env.example` files).
